# AI Daily Digest 详细汇报格式

## 汇报要求

每天中午12:00运行 `ai-daily-digest` skill，生成的报告需要达到以下详细程度：

### 汇报结构

1. **数据概览**
   - 扫描源数、抓取文章数、时间范围、精选篇数
   - 分类分布表格

2. **TOP 3 重磅文章** (每个都要包含)
   - 评分和分类标签
   - 基本信息（来源、时间）
   - **核心信息**: 模型参数、版本、性能数据等
   - **详细要点**: 至少3-5个要点
   - **技术细节**: 攻击手法、算法原理等
   - **为什么重要/值得读**: 行业影响、警示意义
   - 相关链接

3. **每个类别下的文章** (每个都要包含)
   - 评分和分类标签
   - **核心讨论/要点**: 至少2-3个
   - **案例详情**: 具体例子和数据
   - **引发的思考**: 哲学思考、未来影响等

4. **数据汇总**
   - 表格形式展示关键指标
   - 分类分布饼图或表格

5. **一句话总结**

### 详细程度标准

- **重磅文章**: 500-1000字/篇，包含所有技术细节
- **普通文章**: 200-500字/篇，包含核心要点
- **数据支撑**: 必须包含具体数字、百分比、案例
- **外链引用**: 必须包含原文链接

### 消息格式

使用 Markdown 格式，保持层次分明，便于阅读。

### 发送流程

1. 每天12:00自动运行 `run_digest.sh`
2. 生成报告保存到 `~/.openclaw/digest-YYYYMMDD.md`
3. 虾虾读取报告内容，按上述详细程度整理后发送给用户
